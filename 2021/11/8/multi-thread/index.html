进程与线程的学习记录

进程：正在运行的程序

  • 是系统进行资源分配和调用的独立单位
  • 每一个进程都有它自己的内存空间和系统资源

线程：进程中的单个顺序控制流，是一条执行路径

  • 单线程：一个进程只有一条执行路径
  • 多线程：一个进程有多条执行路径

多线程的实现方式

  • 继承Thread类
  1. 自定义类继承Thread类
  2. 重写run()方法
  3. 创建自定义对象
  4. 启动线程（用start()方法启动）

Thread类中获取和设置线程名称的方法：
　　String getName()：返回此线程的名称
　　可以用静态方法Thread.currentThread()返回当前正在执行的线程对象的引用
　　void setName(String name)：更改此线程的名称为name
　　也可以在创建线程对象的时候直接调用父类的带参构造方法

线程调度

Java使用的是抢占式调度模型，优先级高的线程能获取更多的CPU时间片，相同则随机
Thread类中获取和设置线程优先级的方法：
　　public final int getPriority()：返回此线程的优先级 // 默认优先级是5，优先级范围1-10
　　public final void setPriority(int newPriority)：更改此线程的优先级

线程控制

　　static void sleep(long millis)：使当前正在执行的线程停留（暂停执行）指定的毫秒数
　　void join：等待这个线程死亡
　　void setDaemon(boolean on)：将此线程标记为守护线程，当运行的线程都是守护线程时，Java虚拟机将退出 　　

  • 实现Runnable接口（优势：避免了单继承的局限性，解耦合）
  1. 自定义类实现Runnable接口
  2. 重写run()方法
  3. 创建自定义对象
  4. 创建Thread类对象，将自定义对象作为构造参数
  5. 启动线程

线程同步

　　把多条语句操作共享数据的代码锁起来，让任意时刻只能有一个线程执行即可，可以使用同步代码块/同步方法/Lock锁来实现

  1. 同步代码块：
  2. 同步方法：（等价于同步代码块锁 this对象）
  3. Lock锁
    Lock实现提供比synchronized方法和语句更广泛的锁定操作
    构造方法：（Lock是接口不能实例化，采用实现类ReentrantLock来实例化）
    　　ReentrantLock()：无参构造，创建一个ReentrantLock实例
    获得锁和释放锁的方法：
    　　void lock()：获得锁
    　　void unlock()：释放锁
    小贴士：
    同步静态方法标签是先静态后方法，即static synchronized，同步静态方法锁的是 类名.class

优势：解决了多线程的数据安全问题
劣势：当线程很多时，每个线程都会去判断同步锁，耗费资源，降低程序运行效率

线程同步的类：
Stringbuffer、Vector、HashTable

多线程的生产者消费者模式

Object类提供了等待和唤醒的方法：
　　void wait()：导致当前的线程等待，直到另一个线程调用对象的notify()方法或者notifyAll()方法
　　void notify()：唤醒正在等待对象监视器的单个线程
　　void notifyAll()：唤醒正在等待对象监视器的所有线程
